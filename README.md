Task Management Web Application
1 项目概述
本项目实现了一个完整的前后端分离 Web 应用系统，用于帮助用户在线管理个人任务。用户可以通过网页注册账号并登录系统，在登录成功后进入任务管理页面，对自己的任务进行添加、查看和删除操作。系统通过前端界面与后端服务器进行数据交互，并将数据存储在数据库中。本项目的目标是实现一个包含用户认证、数据存储以及基本 CRUD 操作的完整网络应用，从而展示现代 Web 应用开发的基本流程。

2 技术框架选择
在本项目中，前端部分选择使用 React 框架进行开发。React 通过组件化的方式构建用户界面，使页面结构更加清晰，同时提高代码的可维护性和复用性。后端部分使用 Node.js 和 Express 框架搭建服务器，用于处理用户请求并提供 API 接口。数据库方面使用 MongoDB Atlas 云数据库来存储用户信息和任务数据，从而实现数据的持久化。

3 前端界面设计
前端部分主要负责用户界面的展示以及用户交互。本项目设计了注册页面、登录页面以及任务管理页面。用户可以在注册页面创建账号，在登录页面进入系统，并在任务管理页面中管理自己的任务。前端使用组件化设计，例如任务列表组件和任务显示组件，从而使代码结构更加清晰，同时也方便后续扩展和维护。

4 后端系统实现
后端服务器使用 Node.js 和 Express 框架开发，主要负责处理前端发送的请求以及数据库操作。系统实现了用户注册、用户登录以及任务管理相关接口。在用户注册时，系统会将用户信息保存到数据库，并对用户密码进行加密处理，从而提高系统安全性。用户登录成功后，可以通过 API 接口对任务数据进行创建、查询和删除操作。

5 数据库设计
本项目使用 MongoDB Atlas 作为数据库系统。数据库主要包含两个数据模型，分别是用户模型和任务模型。用户模型用于存储用户账号信息，例如用户名、邮箱和密码等数据。任务模型用于存储用户创建的任务内容，例如任务标题、任务状态以及任务所属用户。通过这种数据结构，可以保证每个用户只能够访问和管理自己的任务数据。

6 前后端数据交互
在系统运行过程中，前端和后端通过 RESTful API 进行数据通信。前端页面使用 Axios 向后端服务器发送 HTTP 请求，例如获取任务列表或添加新任务。服务器接收到请求后会进行数据处理，并与 MongoDB 数据库进行交互。处理完成后，服务器将结果返回给前端，前端根据返回的数据更新页面内容，从而实现数据的实时更新和同步。

7 系统测试与部署
在项目开发完成后，对系统进行了基本测试，确保用户注册、登录、添加任务以及删除任务等功能可以正常运行。本项目已经部署到云平台，其中前端应用部署在 Vercel 平台，后端服务器部署在 Render 平台，数据库使用 MongoDB Atlas。通过这种方式，用户可以通过浏览器直接访问系统并使用相关功能。

8 项目运行说明与总结
如果需要在本地运行本项目，可以分别进入 client 和 server 文件夹安装依赖并启动项目。首先进入 server 文件夹运行 npm install 并启动服务器，然后进入 client 文件夹运行 npm install 和 npm run dev 启动前端应用。启动完成后即可在浏览器中访问系统并使用相关功能。通过本项目的开发，我完成了一个完整的全栈 Web 应用系统，并学习了如何将前端、后端和数据库结合在一起进行开发，同时也了解了 Web 应用从开发到部署的完整流程。
